Program Description:
-------------------
Index_Version1:
	The given collection is processed to remove the punctuation and stop words. For each word/token lemma is found using Standford (version 3.4.1) lemmatizer. For each token, its occurance (DocId), term frequency in the given doc is found. During this process, maximum term frequency is also determined. Document length of each document is also stored. 

Index_Version2:
	This is similar to the above method except that here instead of lemma, stemming of the token is done here.
	Other fields are stored similarly as above.

Index_Version1_Compressed:
	Here dictionary is compressed using block Coded technique. Since in the previous steps we have the dictionary, we use that loop over the lemmas. For each lemma its length followed by the lemma is found. All the terms are concatenated. Given k-sized block, find the size of k-number of lemmas combined in (len(lemma),lemma) format. This will be the pointer that'll be stored along with posting list.
	For all the numbers in the posting list and term pointer, gamma codes are used to compress. After compressing k-number of posting lists the encoded term pointer is appended.

Index_Version2_Compressed:
	The dictionary is comrpessed using front Coded compression technique. For k-sized block, find the longest common prefix. The format used to store is (length of 1st word,longest_common_prefix,*,remaining portion of 1st word after longest_common_prefix, length(2nd word)-length(longest_common_prefix),$,portion of 2nd word after longest_common_prefix, length(3rd word)-length(longest_common_prefix),$,portion of 3rd word after longest_common_prefix,...) - the process continues for rest of the words in the block. Term pointer is the length the format shown above for each block which is stored and later appended with the posting list.
	Posting list compression is similar to Index_Version1_Compressed.
